Step-by-Step Guide: Google Cloud Setup for Gmail Sync
1. Create / Select a Google Cloud Project
1. 
Go to https://console.cloud.google.com/
2. 
Click the project dropdown at top → New Project (or select existing)
3. 
Name it (e.g. CRM-Backend) → Create
2. Enable the Gmail API
1. 
From the project dashboard, go to APIs & Services → Library
2. 
Search for "Gmail API"
3. 
Click on it → Enable
3. Configure the OAuth Consent Screen
1. 
Go to APIs & Services → OAuth consent screen
2. 
Choose External (even if internal) → Create
3. 
Fill in:
- 
App name: CRM-Backend
- 
User support email: your email
- 
Developer contact info: your email
4. 
Click Save and Continue
5. 
Scopes: Click Add or Remove Scopes, search for gmail.readonly, select https://www.googleapis.com/auth/gmail.readonly → Add → Update → Save and Continue
6. 
Test users: Add your own email → Save and Continue
7. 
Back to dashboard → Publish App → Confirm
4. Create OAuth 2.0 Credentials
1. 
Go to APIs & Services → Credentials
2. 
Click + Create Credentials → OAuth client ID
3. 
Application type: Web application
4. 
Name: CRM-Backend-Client
5. 
Authorized JavaScript origins: https://your-frontend-domain.com
6. 
Authorized redirect URIs — YOU MUST ADD BOTH:
https://your-backend-domain.com/api/v1/auth/google/callback
https://your-backend-domain.com/api/v1/email/auth/callback
7. 
Click Create
5. Copy Credentials to .env
A modal will show your Client ID and Client Secret. Copy them into your .env:
GOOGLE_CLIENT_ID=935197190890-xxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-xxxxxxxxxxxx
GOOGLE_REDIRECT_URI=https://your-backend-domain.com/api/v1/auth/google/callback
GOOGLE_EMAIL_REDIRECT_URI=https://your-backend-domain.com/api/v1/email/auth/callback
ORIGIN=https://your-frontend-domain.com
6. Frontend Integration Flow
In your frontend app:
// Step 1: Get Gmail auth URL from your backend
const res = await fetch('https://your-api.com/api/v1/email/auth', {
  credentials: 'include'
});
const { data } = await res.json();
// Step 2: Redirect user to Google
window.location.href = data.url;
// Step 3: User authorizes → Google redirects back to:
//   https://your-backend.com/api/v1/email/auth/callback?code=xxx&state=userId
//   → Backend stores tokens, then redirects frontend to:
//   https://your-frontend.com?gmail=connected
// Step 4: Trigger sync
await fetch('https://your-api.com/api/v1/email/sync', {
  method: 'POST',
  credentials: 'include'
});
// Step 5: Fetch messages
const messages = await fetch('https://your-api.com/api/v1/email/messages', {
  credentials: 'include'
});
Important Notes
- 
Refresh tokens only work on first auth — the GET /email/auth endpoint includes prompt: 'consent' to force a new refresh token every time. If a user re-connects, they'll get a fresh refresh token.
- 
Test locally — for local development, use localhost in the URIs and add both localhost redirect URIs to Google Cloud Console alongside your production ones (you can have multiple).
